个人博客搭建记录

发表于2026-07-23|更新于2026-07-24
|浏览量:

概述

采用的方案是hexo+butterfly主题美化，编写博客用的软件是测试版本的typora（有米了搞个正式版的）

开始

首先是git和Node.js的安装，这些网上教程都很多。安装完成后，个人这里用的是挂载在github pages的方式。所以还得完成git绑定用户名，邮箱，SSH key。

然后是github pages的实现，在github上新建一个项目，命名为username.github.io，仓库是公开的，不添加md。

下一步是hexo的安装，进入到博客所在的目录并打开git bash，运行npm install -g hexo 在安装完成后进行hexo文件夹的创建 hexo init(如果已有博客在双电脑协助就不需要这一步) ，随后运行

为了更方便的推送到github pages上，我们使用 hexo-deployer-git 这个插件，输入npm install hexo-deployer-git --save下载插件，随后在_config.yml文件中添加

deploy: type: git repo: <你的仓库地址> # 例如：https://github.com/用户名/用户名.github.io.git branch: [分支名] # 例如：gh-pages 或 main

部署完成后就可以使用hexo d来推送博客到我们自己的网站了。

在备份我们博客源文件的时候需要单独使用一个仓库，由于我们使用的butterfly主题在git上有个专门的库。所以直接push是会有警告的，这里的解决方法是使用git的子模块。这样处理可以随时拉去最新版的butterfly。

git rm --cached themes/butterfly（撤销添加）

rm -rf themes/butterfly(当初删除原先的文件夹)

git submodule add https://github.com/jerryc127/hexo-theme-butterfly.git themes/butterfly(以子模块的形式添加主题)
